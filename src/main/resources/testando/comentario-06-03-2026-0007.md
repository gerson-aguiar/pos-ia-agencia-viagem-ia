Nesse primeiro momento estou explorando: olhando exemplos, reproduzindo experimentos e tentando entender, enquanto minha mente processa esse novo universo que está se abrindo para o desenvolvimento de software.

**Meu projeto de estudos:** Criar um agente de agência de viagem.
**Objetivo:** Ser capaz de responder perguntas comuns sobre viagens e pacotes turísticos.

**Stack:** Backend em Quarkus ou Spring Boot, usando Java + LangChain4j + EasyRAG.

**LLM:** Ollama

* **Agente:** mistral:7b-instruct-q4_0
* **Obs:** Inicialmente eu pretendia usar o `gpt-oss:20b`, mas ao tentar executar descobri que meus 16GB de RAM (rodando via WSL no Windows 11) não seriam suficientes.

**Primeiras impressões:**
Nesse pequeno experimento criei um arquivo simples com informações de dois pacotes de viagem. Bastou configurar onde o EasyRAG poderia encontrar os arquivos de knowledge base, e já foi possível fazer perguntas via requisição HTTP POST para a aplicação.

O resultado foi bem interessante: a aplicação respondeu às perguntas de forma “profissional” e relativamente humanizada.

Algo que achei particularmente incrível foi o comportamento **não determinístico**.
Para a mesma pergunta, com a mesma requisição, o agente respondeu com textos diferentes — no final transmitindo o mesmo significado, mas com estruturas e tokens distintos.

Isso é muito irado.

Sobre o setup do backend em si, é relativamente simples de fazer. Subir o Ollama localmente também foi bem tranquilo.
